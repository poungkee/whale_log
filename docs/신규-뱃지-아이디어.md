## 🔧 구현 시 체크리스트

각 신규 뱃지마다:
1. `badge-definitions.ts`에 정의 추가 (`key`, `nameKo`, `descriptionKo`, `icon`, `category`, `isHidden`, `sortOrder`)
2. `badges.service.ts`에 트리거 로직 작성 (어느 액션 시점에 검사할지)
3. 운영 DB seed 동기화 (`seedBadges()` 호출 또는 SQL UPDATE)
4. 뱃지 이미지(PNG) 생성 → `surf-wave-mobile/assets/images/badges/` + `surf-wave-frontend/public/badges/`
5. 모바일 압축 스크립트 실행 (`D:/workspace/scripts/optimize-badges/optimize.js`)

### ⚠️ 구현 비용이 추가로 드는 것

| 뱃지 | 추가 작업 |
|---|---|
| `BIRTHDAY_WAVE` | User 엔티티에 `birthday DATE` 컬럼 추가 + 마이그레이션 + 회원가입/마이페이지 폼 수정 |
| `BIG_WAVE_RIDER`, `HEAD_HIGH` | 다이어리 작성 시 forecast 스냅샷 저장 필요 (현재 저장 여부 확인) |
| `OFFSHORE_HUNTER` | 풍향 vs 해안선 방향 비교 로직 (스팟별 facing 방향 데이터 필요) |
| `TIDE_MASTER` | 조석 데이터 백엔드 추가 (현재 미구현) |
| `VOTE_REBEL` | 투표 결과 vs 다수 의견 비교 로직 |
| `GUIDE_REREADER` | 가이드 읽음 카운트 누적 (현재 set 방식인지 확인 필요) |

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| 백엔드 | NestJS + TypeORM + PostgreSQL 15 |
| 프론트엔드 | React 18 + Vite + TypeScript + Tailwind CSS 4 |
| DB | Docker `surfwave-postgres` |
| 인증 | JWT 자체 인증 (Firebase → 전환 완료) + Google/Kakao 소셜 로그인 |
| 스팟 수 | 124개 (한국 23개 + 발리 101개) |
| 테스트 | 303개 통과 (5 suites) |
| 데이터 소스 | Open-Meteo Marine + Weather API (30분 크론 자동 수집) |

## 4. 경쟁 앱 비교

### 4.1 주요 경쟁자

| 항목 | Surf Wave (우리) | Surfline | WSB FARM |
|------|-------------------|----------|----------|
| 파도 예보 | Open-Meteo (무료) | 자체 모델 (정밀) | 기상청+자체 |
| **개인화 추천** | **레벨+보드별 맞춤** | 단순 등급 | 없음 |
| **Hints (왜 좋은지/나쁜지)** | **한국어 상세 설명** | 영어 간략 | 없음 |
| **다이어리** | **보드/만족도/캘린더/사진** | 없음 | 없음 |
| **자세 감지** | **MediaPipe 실시간** | 없음 | 없음 |
| **서핑 가이드** | **4카테고리 19항목** | 없음 | 없음 |
| 라이브캠 | 미구현 | 핵심 기능 (유료) | 핵심 기능 |
| 커뮤니티 | 시작 단계 | 없음 | 활발 |
| 지도 | 클러스터링 있음 | 있음 | 있음 |
| 스팟 수 | 124개 (한국+발리) | 글로벌 | 한국 집중 |
| 가격 | 무료 | 프리미엄 유료 | 무료 |

### 4.2 차별화 포인트 (우리만 가진 것)

```
1. "나한테 맞는 파도인지" 개인화 추천
   → 레벨(초급~전문가) + 보드(9종류)별 맞춤 점수
   → 다른 앱: 모든 사용자에게 같은 등급 표시

2. "왜 좋은지/나쁜지" Hints 시스템
   → "오프쇼어 바람이라 파도면이 깨끗해요" 같은 한국어 설명
   → 다른 앱: 숫자만 보여주고 해석은 사용자 몫

3. AI 자세 감지 (세계 최초 서핑앱 적용)
   → 실시간 스켈레톤 + 측정값 표시
   → Surfline: 안 하는 것 (비즈니스 모델 밖)
   → WSB: 못 하는 것 (기술력 부족)

4. 서핑 다이어리 + 성장 추적
   → 사진 첨부, 파도 데이터 자동 매칭, 통계
   → 다른 앱: 예보만 제공, 기록 기능 없음

5. 초보 서핑 가이드
   → 바다 규칙, 안전 상식, 용어, 기본 자세
   → 다른 앱: 이미 잘 타는 사람 대상
```

### 4.3 약점 (솔직한 평가)

```
1. 예보 정확도
   → Open-Meteo는 offshore 데이터 기반 → 연안 실제 파도와 차이
   → Surfline/WSB는 현장 보정 + 경험적 모델 보유

2. 라이브캠 없음
   → 한국 서퍼가 앱 켜는 가장 큰 이유: "지금 파도 어때?"
   → 유튜브 임베드로 일부 대체 가능하나 커버리지 제한

3. 커뮤니티 비어있음
   → WSB 네이버 카페가 이미 한국 서핑 커뮤니티 장악
   → 사용자 기반 없이 커뮤니티 기능만 넣으면 빈 방

4. 스팟 정보 깊이
   → 로컬 팁, 위험요소, 주차/편의시설 등 현장 정보 부족
   → 사용자 기여 데이터가 쌓여야 해결 가능
```

### 4.4 전략적 포지셔닝

```
Surfline = "프로 서퍼의 도구" (예보 정확도 + 캠으로 승부)
WSB FARM = "한국 서퍼의 커뮤니티" (CCTV + 카페로 승부)
Surf Wave = "초보 서퍼의 코치" (개인화 + 교육 + 성장으로 승부)

→ 타겟이 다르기 때문에 공존 가능
→ 서핑 인구의 대부분은 초보~중급이므로 시장은 충분
→ 핵심 가치: "서핑 시작하고 혼자 늘 수 있는 앱"
```

## 5. 기술 아키텍처 요약

### 5.1 서핑 적합도 계산 엔진 (v1.5)

```
STEP 1: 하드블록 안전 필터
  → 풍속 35km/h+, 파고 1.2m+(초급), 수온 10°C 이하 등 → BLOCKED

STEP 2: 5개 항목 Fit 점수 (각 0~10)
  → waveFit(30%) + windSpeedFit(20%) + windDirFit(20%)
    + swellFit(10%) + periodFit(10%)
  → 보드별 파고 보정 (롱보드: 작은 파도 OK, 숏보드: 큰 파도 필요)

STEP 3: 가중합 → rawSurfRating (0~10)
  → 플랫 파도(0m) 페널티: 최대 1.0점
  → False Positive 보정
  → Grace Zone (경계선 오프쇼어 보호)

STEP 4: 레벨별 판정
  → PASS / WARNING / BLOCKED + 한국어 추천 문구
```

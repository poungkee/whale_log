## 2. 지도 API 조사 결과

### 2.1 무료 옵션 비교표

| 옵션 | 비용 | 신용카드 | 월 한도 | 벡터 타일 | 3D 지형 | React 라이브러리 | 추천도 |
|------|------|----------|---------|-----------|---------|-----------------|--------|
| **MapLibre + OpenFreeMap** | 무료 | 불필요 | 무제한 | ✅ | ✅ | `react-map-gl` | ★★★★★ |
| **Leaflet + OpenStreetMap** | 무료 | 불필요 | 무제한* | ❌ (래스터) | ❌ | `react-leaflet` | ★★★★ |
| **Kakao Maps API** | 무료 | 불필요 | 일 30만건 | ❌ (래스터) | ❌ | `react-kakao-maps-sdk` | ★★★★ |
| Mapbox | 프리미엄 | **필요** | 월 5만 로드 | ✅ | ✅ | `react-map-gl` | ★★★ |
| Google Maps | 프리미엄 | **필요** | 월 1만/SKU | ✅ | ✅ | `@react-google-maps/api` | ★★ |
| Naver Maps | **유료** | 필요 | 무료 티어 폐지 (2025.7~) | - | - | `react-naver-maps` | ✗ |

> *OSM 타일은 공정 사용 정책 (초당 2요청 이하) 적용, 월간 총량 제한은 없음

### 2.2 추천 조합

**1순위: MapLibre GL JS + OpenFreeMap**
- 완전 무료, API 키 불필요, 등록 불필요
- 벡터 타일 → 해안선 부드럽게 렌더링, 줌 레벨 무관
- 커스텀 스타일링 가능 (바다/해안 강조, 내륙 약화)
- 3D 지형 표시 가능 → 해안 지형 시각화
- `react-map-gl` 라이브러리로 React 통합 우수
- 타일 스타일: `liberty`, `positron`, `bright` 3종 제공

```tsx
// MapLibre + OpenFreeMap 사용 예시
import Map, { Marker, Popup } from 'react-map-gl/maplibre';
import 'maplibre-gl/dist/maplibre-gl.css';

<Map
  initialViewState={{ longitude: 129.0, latitude: 35.1, zoom: 10 }}
  mapStyle="https://tiles.openfreemap.org/styles/liberty"
>
  <Marker longitude={129.13} latitude={35.15}>
    <div className="surf-marker">해운대</div>
  </Marker>
</Map>
```

**보조: Kakao Maps API (한국 특화)**
- 한국어 지명, POI 정확도 최고
- 일 30만건 무료 (사실상 무제한)
- 카카오 개발자 계정만 필요 (무료)

### 2.3 Naver Maps 유료 전환 주의

2025년 7월 1일부터 Naver Maps API 무료 티어가 **완전 폐지**되었습니다. 모든 사용량에 대해 과금되므로 무료 프로젝트에서는 사용 불가합니다.

## 3. 위치 기반 검색 조사 결과

### 3.1 브라우저 Geolocation API

| 항목 | 상세 |
|------|------|
| **비용** | 완전 무료 (브라우저 내장) |
| **API 키** | 불필요 |
| **월 한도** | 없음 |
| **HTTPS 필수** | ✅ (localhost 제외) |
| **사용자 권한** | 필요 (브라우저 권한 팝업) |

### 3.2 정확도

| 소스 | 일반적 정확도 | 비고 |
|------|-------------|------|
| GPS (모바일, 실외) | 3~10m | 해변에서 사용 시 최적 |
| WiFi (도심) | 20~75m | 서핑 스팟 찾기에 충분 |
| 셀 타워 | 100~3,000m | 대략적 위치 파악 가능 |
| IP 주소 (데스크탑) | 1~50km | 도시 레벨, 대략적 지역 판별만 |

### 3.3 구현 코드 예시

```typescript
// 브라우저 위치 가져오기 (무료)
navigator.geolocation.getCurrentPosition(
  (position) => {
    const { latitude, longitude, accuracy } = position.coords;
    // accuracy: 미터 단위 (95% 신뢰도)
    // → 이 좌표로 가까운 서핑 스팟 계산
  },
  (error) => { /* 위치 권한 거부 등 에러 처리 */ },
  { enableHighAccuracy: true, timeout: 10000, maximumAge: 0 }
);
```

### 3.4 활용 가능 기능 (추후 결정)

- **가까운 서핑 스팟 찾기**: "내 근처 스팟" 기능
- **거리/소요시간 표시**: 현재 위치 → 스팟까지 직선거리
- **지도 중심 자동 설정**: 한국에 있으면 한국 지도, 발리에 있으면 발리 지도
- **지역별 자동 필터링**: 위치 기반 추천 스팟 우선 표시

## 5. 해양 API 정확도 및 교차 검증

### 5.1 현재 사용 중인 API 분석

**Open-Meteo Marine API** (메인 데이터 소스)

| 항목 | 상세 |
|------|------|
| **비용** | 완전 무료 (비상업 ~10,000건/일) |
| **API 키** | 불필요 |
| **기반 모델** | ECMWF WAM + NOAA GFS Wave (자동 선택) |
| **해상도** | 0.25도 (~25km) |
| **예보 기간** | 7일 (GFS 기준 최대 16일) |
| **업데이트 주기** | 6시간마다 (00Z, 06Z, 12Z, 18Z) |
| **수집 데이터** | 파고, 파주기, 파방향, 스웰(높이/주기/방향), 풍속, 풍향, 돌풍 |
| **미수집 데이터** | 조석(tide), 수온, 기온, 날씨 상태 |
| **커버리지** | 전 세계 (한국 ✅, 발리 ✅) |

**정확도 평가:**
- ✅ Surfline 등 상용 앱과 **동일한 기반 모델** (GFS Wave, ECMWF WAM) 사용
- ✅ 일반적인 파도 예보에는 충분한 정확도
- ⚠️ 합성 파고/주기 값은 평균치 → 복합 스웰 상황에서 부정확 가능
- ⚠️ 25km 해상도 → 근접 스팟 간 미세 차이 반영 어려움
- ⚠️ 조석 데이터 미수집 → 리프 브레이크 (발리 대부분) 정확도 한계

### 5.2 교차 검증용 무료 API 비교

| API | 무료 한도 | 파도 | 스웰 | 바람 | 조석 | 한국 | 발리 | 통합 난이도 |

| **Open-Meteo** (현재) | ~10,000/일 | ✅ | ✅✅✅ | 별도 API | △ | ✅ | ✅ | 매우 쉬움 |
| **Stormglass** | **10/일** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | 쉬움 |
| **NOAA GFS/WW3** | 무제한 | ✅ | ✅ | ✅ | ❌ | ✅ | ✅ | 어려움 (GRIB2) |
| **Copernicus (CMEMS)** | 무제한 | ✅ | ✅ | ✅ | ❌ | ✅ | ✅ | 보통 (Python) |

### 5.3 사용하면 안 되는 API

| API | 이유 |
|-----|------|
| ❌ **Windy** | 무료 티어 데이터를 **의도적으로 변조** (randomly shuffled) 해서 제공. 검증 불가 |
| ❌ **Surfline** | 공식 API 없음. 비공식 접근은 언제든 차단 가능. 프로덕션 의존 불가 |
| ❌ **Naver Maps** | 2025.7월부터 무료 폐지 |
| ⚠️ **WorldWeatherOnline** | 60일 체험판 후 유료. 장기 사용 불가 |

### 5.4 추천 교차 검증 전략

```
┌─────────────────────────────────────────────────────┐
│                    매일 (자동)                         │

│  Open-Meteo (메인)          Stormglass (검증)         │

│                                                       │

```

### 6.1 현재 시스템 미구현 필드

DB에 컬럼은 이미 존재하지만 API에서 데이터를 가져오지 않는 항목:

| 필드 | DB 컬럼 | 상태 | 중요도 |
|------|---------|------|--------|

| 수온 | `waterTemperature` | 미구현 | 🟡 높음 |
| 기온 | `airTemperature` | 미구현 | 🟢 보통 |
| 날씨 상태 | `weatherCondition` | 미구현 | 🟢 보통 |

### 6.2 기능 우선순위 분석

#### 🔴 높은 우선순위 (서핑앱 필수)

| 기능 | 이유 | 난이도 |
|------|------|--------|
| **조석 데이터 추가** | 발리 리프 브레이크는 타이드에 따라 서핑 가능 여부가 완전히 달라짐. Green Bowl은 0.0~0.2m일 때만 서핑 가능 | 중 |
| **스팟 대폭 확장 (97개)** | 16개 → 97개. 좌표/난이도/타입 데이터 확보 완료 | 하 (데이터 입력) |
| **지도 통합** | 스팟 위치 시각화. 특히 발리 리프는 지도 없이 찾기 불가능 | 중 |
| **스웰 방향별 스팟 매칭** | 현재 단순 파고 기준 → 스팟별 최적 스웰 방향 매칭 시 추천 정확도 대폭 향상 | 중 |

#### 🟡 중간 우선순위 (차별화)

| 기능 | 이유 | 난이도 |
|------|------|--------|
| **수온 데이터** | 웻수트 선택 필수 정보. Open-Meteo에서 제공 가능 | 하 |
| **시간대별 예보 차트** | 현재 "지금" 데이터만 → 24h/7일 그래프 시각화 | 중 |
| **즐겨찾기 스팟** | 자주 가는 스팟 빠르게 확인 | 하 |
| **푸시 알림** | FCM 이미 세팅됨. "내 스팟에 좋은 파도 올 때" 알림 | 중 |
| **시즌 자동 추천** | 발리 건기=서쪽, 우기=동쪽 자동 추천 | 하 |

#### 🟢 낮은 우선순위 (나중에)

| 기능 | 이유 | 난이도 |
|------|------|--------|

# 서핑 파도 앱 - 기능 추가 백로그

> 1차 구현에서 제외된 기능들을 우선순위별로 정리한 문서입니다.
> 백엔드 API는 이미 준비되어 있으며, 프론트엔드 연동만 필요한 항목이 대부분입니다.

---

## 우선순위 HIGH (사용성에 큰 영향)

### 1. 즐겨찾기 (찜하기) 기능
- **백엔드 API**: `POST /spots/:id/favorite`, `DELETE /spots/:id/favorite`, `GET /spots/favorites`
- **구현 내용**: SpotCard에 하트 아이콘 추가, 탭 또는 필터로 즐겨찾기 스팟만 보기
- **비고**: 로그인 필요 (JWT 인증)

### 2. 오늘의 컨디션 투표 기능
- **백엔드 API**: `POST /spots/:id/vote`, `GET /spots/:id/votes`
- **구현 내용**: 상세 모달에 "오늘 컨디션 어때요?" 투표 UI (좋아요/별로예요)
- **데이터**: `{ perfect, flat, totalVotes, userVote }` 표시
- **비고**: 실제 서퍼들의 피드백으로 예보 정확도 간접 검증 가능

### 3. 지역 필터
- **백엔드 API**: `GET /spots?region=부산` (region 파라미터)
- **구현 내용**: 홈 헤더에 지역 드롭다운 추가 (전체/부산/양양/제주/서귀포/기타)
- **비고**: 97개 스팟에서 관심 지역만 필터링

### 4. 기온 / 수온 / 날씨 표시
- **백엔드 데이터**: `forecast.airTemperature`, `forecast.waterTemperature`, `forecast.weatherCondition`
- **구현 내용**: SpotCard 또는 상세 모달에 기온/수온 아이콘 표시
- **비고**: 수온은 웻슈트 선택에 핵심 정보, 서퍼들에게 매우 중요

### 5. 시간대별 예보 탭 (상세 모달)
- **백엔드 API**: `GET /spots/:id/forecast?hours=24` (시간별 예보)
- **구현 내용**: 상세 모달의 "지금 / 오후 3시 / 내일 아침" 탭 실제 데이터 연동
- **비고**: 현재 대시보드 API는 가장 가까운 1개 예보만 반환 → 별도 API 호출 필요

## 우선순위 MEDIUM (기능 확장)

### 6. 스팟 검색
- **백엔드 API**: `GET /spots?search=송정` (name 검색)
- **구현 내용**: 홈 헤더의 검색 버튼 → 검색 입력창 + 실시간 필터링
- **비고**: 현재 검색 버튼은 UI만 존재

### 7. 주간 예보 차트
- **백엔드 API**: `GET /spots/:id/forecast/weekly`
- **구현 내용**: 상세 모달에 7일간 파고/풍속 추이 라인 차트 (recharts 활용)
- **데이터**: `{ date, minWaveHeight, maxWaveHeight, avgWavePeriod, avgWindSpeed }`
- **비고**: recharts 이미 설치됨, 미사용 상태

### 8. 근처 스팟 (GPS 기반)
- **백엔드 API**: `GET /spots/nearby?lat=&lng=&radius=50`
- **구현 내용**: GPS 위치 요청 → 가까운 순서로 스팟 정렬 또는 별도 탭
- **비고**: 지도(Map) 탭과 연동 가능

### 9. 돌풍(Gust) 정보 표시
- **백엔드 데이터**: `forecast.windGusts` (km/h)
- **구현 내용**: 상세 모달 바람 섹션에 "돌풍 최대 Xkm/h" 추가
- **비고**: 안전에 중요한 정보, BLOCKED 판정에도 사용됨

### 10. 스팟 사진 / 이미지
- **백엔드 데이터**: `spot.imageUrl`
- **구현 내용**: SpotCard 상단에 스팟 대표 이미지 표시
- **비고**: 이미지 업로드 API (`POST /upload`) 존재, 이미지 데이터 수집 필요

## 우선순위 LOW (부가 기능)

### 11. 스팟 상세 정보 섹션
- **백엔드 데이터**: `spot.breakType`, `spot.season`, `spot.amenities`, `spot.description`
- **구현 내용**: 상세 모달 하단에 스팟 정보 섹션 (파도 유형, 최적 시즌, 편의시설)
- **비고**: 초보자에게 유용한 교육 정보

### 12. 스팟 평점 / 리뷰
- **백엔드 데이터**: `spot.rating`, `spot.ratingCount`
- **구현 내용**: SpotCard에 별점 표시 (예: 4.2/5.0 (23))
- **비고**: 현재 모든 스팟 rating=0, 리뷰 시스템 구현 필요

### 13. 지도(Map) 탭 구현
- **백엔드 데이터**: `spot.latitude`, `spot.longitude`
- **구현 내용**: Kakao Map 또는 Google Maps 연동, 스팟 핀 표시
- **비고**: BottomNav에 지도 탭 이미 존재 (플레이스홀더)

### 14. 피드(Feed) 탭 - 커뮤니티
- **백엔드 API**: `/posts`, `/comments`, `/questions`, `/answers`
- **구현 내용**: 서핑 세션 공유, 사진, 질문/답변 피드
- **비고**: BottomNav에 피드 탭 이미 존재 (플레이스홀더)

### 15. 파도 방향 표시
- **백엔드 데이터**: `forecast.waveDirection` (도)
- **구현 내용**: 상세 모달에 파도 방향 화살표 또는 나침반 표시
- **비고**: 스웰 방향과 별개의 파도 방향

### 16. 알림 기능 (푸시)
- **백엔드**: Firebase FCM Admin SDK 유지 중
- **구현 내용**: 즐겨찾기 스팟의 컨디션이 좋을 때 푸시 알림
- **비고**: 서비스 워커 + FCM 토큰 관리 필요
